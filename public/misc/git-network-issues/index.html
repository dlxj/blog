国内使用 Git 的网络问题

February 22, 2025
  • 我的系统：Windows 11

当时遇到的情况是，已经使用 VPN ，浏览器可以顺利上网，但在终端使用 git 相关命令总遇到类似这样的报错：

Could not resolve host: raw.githubusercontent.com

caused by: [28] Timeout was reached (Failed to connect to github.com port 443 after 21036 ms: Could not connect to server)

后来得知是要进行一些设置，终端才能像浏览器一样使用 VPN。我进行以下操作后，就能顺利连接 git 了！（特别感谢青羽提供的帮助）

设置 VPN

特别地，如果无法开启服务模式，可参考这篇Clash for Windows 手动安装 Service Mode解决。

添加 hosts

打开C:\Windows\System32\drivers\etc，找到hosts文件。要修改这个文件需要管理员权限，一个简单的办法是，把它复制到桌面，用记事本打开这个文件，在最后一行写入185.199.108.133 raw.githubusercontent.com，保存，替换原来那个，重启电脑。

测试

打开终端，用ping测试终端是否已经像浏览器一样使用 VPN。
